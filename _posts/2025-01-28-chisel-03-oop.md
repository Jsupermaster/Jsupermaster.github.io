> 上一篇我们学完了Scala函数式编程的基础，本篇正式进入Scala面向对象编程的世界——这是Chisel硬件模块组织的核心基础。

## 3.1 类和对象

### 3.1.1 类

类是用`class`开头的代码定义，定义完成后可以用`new` + 类名的方式构造一个对象，对象的类型是这个类。

- 类中定义的`var`和`val`类型变量称为**字段**
- 用`def`定义的函数称为**方法**

字段也称为实例变量，因为每个被构造出来的对象都有自己的字段，但所有的对象公用同样的方法。使用`new`创建的对象可以赋值给一个变量，赋值给用`val`定义的变量则之后不得被赋值给新的对象：

```scala
scala> class Students {
     | var name = "None"
     | def register(n:String) = name = n
     | }
// defined class Students

scala> val stu = new Students
val stu: Students = Students@4fc454d5

scala> stu.name
val res29: String = None

scala> stu.register("Bob")

scala> stu.name
val res30: String = Bob

scala> stu = new Students
-- [E052] Type Error: Reassignment to val stu
```

类的成员都是公共的，在外部都可以被访问。如果不想被其他对象访问，可以在变量前加上`private`关键字。

### 3.1.2 类的构造方法

#### 主构造方法

Scala不同于C++、Python等语言，Scala没有专门的主构造语法。在类名后可以定义若干参数列表用于传递参数，参数会在初始化对象时传给类中的变量。**类内部非字段、非方法的代码都被当做"主构造方法"。**

```scala
scala> class Students(n:String) {
     | val name = n
     | println("A student named" + n + " has been registered.")
     | }
// defined class Students

scala> val stu = new Students("Tom")
A student namedTom has been registered.
val stu: Students = Students@33004de5
```

#### 辅助构造方法

在类的内部使用 `def this(...)` 定义一个辅助构造方法，其第一步必须是调用该类的另一个构造方法，即第一句必须是`this(...)`，要么是主构造方法，要么是之前的另一个辅助构造方法。这种规则让任何构造方法最终都会调用类的主构造方法，使得主构造方法成为类的单一入口。

```scala
scala> class Students(n:String) {
     | val name = n
     | def this() = this("None")
     | println("A student named " + n + " has been registered.")
     | }
// defined class Students

scala> val stu = new Students
A student named None has been registered.
val stu: Students = Students@6f319f62
```

> **注意：** 不能定义两个有歧义的辅助构造函数。Scala不需要定义析构函数。

#### 私有主构造方法

在类的定义时加入`private`关键字，则主构造函数是私有的，外部无法调用主构造函数创建对象，只能使用其他公有的辅助构造方法或者工厂方法：

```scala
scala> class Students private (n:String, m:Int) {
     | val name = n
     | val score = m
     | def this(n:String) = this(n, 100)
     | println(n + "'s score is " + m)
     | }
// defined class Students

scala> val stu = new Students("Bill", 90)     // ERROR: 主构造方法私有

scala> val stu = new Students("Bill")          // OK: 使用辅助构造方法
Bill's score is 100
```

### 3.1.3 重写toString方法

构造对象时Scala解释器打印了类似`Students@45f675a4`的信息，这来自`toString`方法。默认的`toString`打印类名、"@"和十六进制数。可以加上`override`关键字自定义：

```scala
scala> class Students(n:String) {
     | val name = n
     | override def toString = "A student named " + n + "."
     | }
// defined class Students

scala> val stu = new Students("Jia")
val stu: Students = A student named Jia.
```

### 3.1.4 方法重载

与C++类似，Scala支持方法重载，即在类内可以定义多个同名的方法，但参数（主要是参数类型）不一样。函数的真正特征是它的参数，而非函数名或返回类型。**重载是同一个类中的同名方法的不同版本，重写（override）是子类覆盖超类的方法。**

### 3.1.5 类参数

Scala允许在类参数前加上`val`或`var`来修饰，这样可以在类的内部生成一个与参数同名的公有字段。还可以使用`private`、`protected`、`override`来表明字段的权限。

- 如果参数**没有任何关键字**，那么它就仅仅是参数，不是类的成员，只能用于初始化字段赋值，内部无法修改，外部无法访问。
- 加上`val`或`var`后，参数也成为字段，可以从外部访问。

```scala
scala> class Students(val name: String, var score: Int) {
     | def exam(s:Int) = score = s
     | override def toString = name + "'s score is " + score + "."
     | }
// defined class Students

scala> val stu = new Students("Tim", 90)
val stu: Students = Tim's score is 90.

scala> stu.exam(100)
scala> stu.score
val res32: Int = 100
```

### 3.1.6 单例对象与伴生对象

除了使用`new`创建一个对象，也可以使用`object`关键字。但它**没有参数和构造方法，且数量只能有一个**，因此被称为**单例对象**。

如果某个单例对象和某个类同名，则称单例对象是这个类的**伴生对象**，这个类称为这个单例对象的**伴生类**。伴生类和伴生对象必须在同一个文件中，并且可以互相访问对方的所有成员。

```scala
scala> object B {val b = "a singleton object"}
// defined object B

scala> B.b
val res33: String = a singleton object

scala> val x = B
val x: B.type = B$@3c4f0087
```

**重要区别：** 定义一个类就定义了一个类型；但定义单例对象并没有新产生一种类型。每个单例对象的类型都是`object.type`。不同的`object`定义之间并不能互相赋值。

### 3.1.7 工厂对象与工厂方法

如果定义一个专门用来构造某一个类的对象的方法，那么这种方法被称为**工厂方法**，包含这些工厂方法集合的单例对象称为**工厂对象**。一般工厂对象会定义在伴生对象中。使用工厂方法的好处是可以不用直接使用`new`来实例化对象，对外隐藏了类的实现细节。

```scala
class Students(val name: String, var score: Int) {
  def exam(s:Int) = score = s
  override def toString = name + "'s score is " + score + "."
}

object Students {
  def registerStu(name: String, score:Int) = new Students(name,score)
}
```

编译后：

```bash
scalac students.scala
scala -classpath .
```

```scala
scala> import Students._
scala> val stu = registerStu("Tim", 100)
val stu: Students = Tim's score is 100.
```

### 3.1.8 apply方法

`apply`方法是一个特殊名字的方法。如果定义了这个方法，可以显式调用`对象.apply(参数)`，也可以隐式调用`对象(参数)`。

```scala
class Students2(val name: String, var score: Int) {
  def apply(s: Int) = score = s
  def display() = println("Current score is " + score + ".")
  override def toString = name + "'s score is " + score + "."
}

object Students2 {
  def apply(name: String, score: Int) = new Students2(name, score)
}
```

```scala
scala> val stu2 = Students2("Jack", 60)  // 隐式调用伴生对象apply
val stu2: Students2 = Jack's score is 60.

scala> stu2(80)                           // 隐式调用类的apply
scala> stu2.display()
Current score is 80.
```

### 3.1.9 主函数

主函数是Scala程序的唯一入口。必须在某个单例对象中定义一个名为`main`的函数，且该函数只有一个参数，类型为`Array[String]`，返回类型是`Unit`。

```scala
object Start {
  def main(args: Array[String]) = {
    try {
      val score = args(1).toInt
      val s = Students2(args(0), score)
      println(s.toString)
    } catch {
      case ex: ArrayIndexOutOfBoundsException => println("Arguments are deficient!")
      case ex: NumberFormatException => println("Second argument must be a Int!")
    }
  }
}
```

```bash
scala Start.scala students2.scala -- Tom 100
Tom's score is 100.
```

## 3.2 操作符即方法

### 3.2.1 操作符在Scala中的解释

**严格来说，Scala不存在操作符的概念。** 加减乘除、逻辑运算的与或非、比较运算的大于小于等，其实都是定义在`class Int`、`class Double`中的成员方法。在Scala中，**操作符即方法**。

表达式`1 + 2`的真正形式是`1.+(2)`——调用了Int类的`+`方法。

Scala中任何类定义的成员方法都是操作符，且方法调用都能写成操作符的形式：去掉句点符号，并且在方法参数只有一个时可以省略括号。在新版Scala中，中缀调用需要标注`@infix`注解：

```scala
import scala.annotation.infix

class Student {
  @infix
  def exam(score: Int): Unit = {
    println(s"Score: $score")
  }
}
```

### 3.2.2 三种操作符

**前缀操作符：** 只有`+`、`-`、`!`、`~`这四个，操作数只有1个，命名格式为`unary_+`。

```scala
scala> class MyInt(val x:Int) {
     | def unary_! = -x
     | }
// defined class MyInt

scala> val mi = new MyInt(10)
scala> !mi
val res1: Int = -10
```

**中缀操作符：** 左右两边都接收操作数，对应普通的有参方法。**以冒号`:`结尾的操作符，其右操作数是调用该方法的对象。**

```scala
scala> class MyInt2(val x:Int) {
     | def +*(y:Int) = (x + y) * y
     | def +:(y:Int) = x + y
     | }
scala> val mi2 = new MyInt2(10)
scala> mi2 +* 10
val res3: Int = 200

scala> 10 +: mi2       // 冒号结尾，右操作数是调用对象
val res4: Int = 20
```

**后缀操作符：** 写在操作数后面，操作数只有一个，即调用该方法的对象。

### 3.2.3 操作符的优先级和结合性

优先级通过方法名的首个字符比较（从高到低）：

| 首个字符     | 优先级     |
| ------------ | ---------- |
| 所有其他字符 | 最高       |
| */%          |            |
| +-           |            |
| :            |            |
| =!           |            |
| <>           |            |
| &            |            |
| ^            |            |
| \|           |            |
| 所有字母     |            |
| 所有赋值操作符 | 最低     |

**结合性：** 一般情况下从左往右结合，但以冒号结尾的中缀操作符是从右往左结合的。

### 3.2.4-5 预设操作符与对象相等性

Scala预设了常用的算术运算、逻辑运算操作符。`==`和`!=`比较自然相等性（字面值相等），`eq`和`ne`方法比较引用相等性（是否指向同一个对象）。

## 3.3 类继承

### 3.3.1 Scala中的类继承

通过在类的参数列表后面加上关键字`extends`和被继承类的类名，就完成了继承的过程：

```scala
scala> class A {
     | val a = "Class A"
     | }
// defined class A

scala> class B extends A {
     | val b = "Class B inherits from A"
     | }
// defined class B

scala> val x = new B
scala> x.a
val res0: String = Class A
```

### 3.3.2 调用超类的构造方法

Scala只允许主构造方法调用超类的构造方法。语法：

```
class 子类(子类对外接收的参数) extends 超类(子类给超类的参数)
```

```scala
scala> class A(val a: Int)
scala> class B(giveA: Int, val b: Int) extends A(giveA)
scala> val x = new B(10, 20)
scala> x.a
val res2: Int = 10
```

### 3.3.3 重写超类的成员

- 超类中用`private`修饰的私有成员和被子类重写的成员不会被继承。
- 重写时应该在开头加上`override`。
- 用`final`修饰的成员不可被子类重写，`final`修饰的`class`不能被继承。
- **Scala允许超类的无参方法被子类重写为字段**，但字段不能反过来被重写为方法。

### 3.3.4 子类型多态与动态绑定

类型为超类的变量可以指向子类的对象，这称为**子类型多态**。方法的版本是**动态绑定**的——由变量指向的对象来决定调用哪个版本：

```scala
scala> class A { def display() = "I'm A." }
scala> class B extends A { override def display() = "I'm B." }
scala> val x: A = new B
scala> x.display()
val res4: String = I'm B.    // 调用的是B类的方法！
```

### 3.3.5 抽象类

类中包含没有初始化的字段或没有函数体的方法时，这个类是**抽象类**，必须用`abstract`修饰。抽象类无法通过`new`来构造实例对象。子类补全了相关定义后，称子类"实现"了超类的抽象成员。

```scala
scala> abstract class A { val a: Int }
scala> class B(val b: Int) extends A { val a = b * 2 }
scala> val y = new B(1)
scala> y.a
val res5: Int = 2
```

### 3.3.6 多重继承

Scala**没有**多重继承，而是设计了"特质"（trait）来实现相同的功能，规则更加简单明了。

### 3.3.7 Scala类的层次结构

最顶部是抽象类**`Any`**，所有类的超类。`Any`有两个子类：**`AnyVal`**（所有值类的父类，9个基本类型）和 **`AnyRef`**（所有引用类的父类）。**`Nothing`**是所有类的子类，表示空值。**`Null`**是空引用类型。

## 3.4 特质（Trait）

### 3.4.1 什么是特质

特质使用`trait`开头，与单例对象很像：都不能有输入参数，单例对象是具体的，特质是抽象的。两者都不能用`new`实例化。

特质可以被其他类、单例对象和特质**"混入"（mixin）**。Scala只允许继承自**一个**类，但是对特质的混入**数量没有限制**。混入第一个特质使用`extends`，之后的用`with`：

```scala
scala> class A { val a = "Class A" }
scala> trait B { val b = "Trait B" }
scala> trait C { val c = "Trait C" }
scala> object D extends A with B with C
scala> D.a
val res7: String = Class A
scala> D.b
val res8: String = Trait B
scala> D.c
val res9: String = Trait C
```

### 3.4.2 特质的层次

特质也可以继承自其他类或混入其他特质，但有限制条件：**要混入该特质的类/单例对象/特质，它的超类必须是待混入特质的超类，或是待混入特质的超类的子类。**

### 3.4.3 混入特质的简便方法

快速构建一个混入某些特质的实例：

```
new SuperClass with Trait1 with Trait2 ... {definition}
```

这定义了一个**匿名类**，匿名类混入了这些特质。

### 3.4.4 特质的线性化叠加计算

Scala的特质采取一种**线性化**的规则来调用特质中的方法。在特质中`super`调用是**动态绑定**的——直到特质混入某个类或别的特质、有了具体的超类方法，才能确定`super`的行为。

**实现线性叠加的要点：**
1. 在特质中定义同名同参方法，加关键字组合 **`abstract override`**（只能用于特质）；
2. 该特质必须混入某个拥有该方法的具体定义的类中；
3. 需要线性化的类定义时不能立即混入特质；
4. 特质中对方法的定义必须出现`super.方法名(参数)`；

**线性化计算示例：**

```scala
abstract class A { def m(s: String) : String }
class X extends A { def m(s: String) = "X ->" + s }

trait B extends A { abstract override def m(s: String) = super.m("B ->" + s) }
trait C extends A { abstract override def m(s: String) = super.m("C ->" + s) }
trait D extends A { abstract override def m(s: String) = super.m("D ->" + s) }
trait E extends C { abstract override def m(s: String) = super.m("E ->" + s) }
trait F extends C { abstract override def m(s: String) = super.m("F ->" + s) }

class G extends X { override def m(s: String) = "G ->" + s }

val x = new G with D with E with F with B
@main def test() = println(x.m("End"))
```

线性化序列计算：`G -> B -> F -> E -> C -> D -> A`，执行结果：**`G -> D -> C -> E -> F -> B -> End`**。

**线性化是Scala特质系统最精妙的部分，实现了对同一方法的多层"装饰"，在Chisel中被大量用于构建硬件模块的可叠加功能。**
