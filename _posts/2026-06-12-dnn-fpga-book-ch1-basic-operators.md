> 📖 **本书来源：**《深度神经网络FPGA设计与实现》，孙其功 等编著，西安电子科技大学出版社，2020年

这篇笔记整理自《深度神经网络FPGA设计与实现》中我认为最值得单独拎出来复习的一部分，也就是**神经网络基础算子**。相比书里前面较多的背景铺垫，这一章更接近真正做加速器实现时会反复遇到的问题：卷积怎么算、反卷积和上采样是什么关系、池化与激活各自解决什么问题、为什么 BN 和 Shortcut 会显著影响网络结构设计。

先说结论：这本书整体并不是我最推荐的 FPGA 神经网络入门书，原因在于前面几章对平台、框架和发展史的介绍占比偏大，而真正落到硬件实现细节时又不够深入，尤其缺少完整工程参考。但如果只是把它当作一份**神经网络算子与 FPGA 加速问题的概览材料**，这一章依然有整理价值。

## 1.1 卷积算子

卷积是绝大多数视觉网络最核心的计算单元。书里对卷积的讲解主要围绕五个参数展开：

| 参数名称 | 参数符号 |
| :--- | :---: |
| 输入特征图尺寸 | `I` |
| 卷积核尺寸 | `K` |
| 滑动步长 | `S` |
| 边缘填充像素数 | `P` |
| 输出特征图尺寸 | `O` |

输出特征图尺寸的计算公式是卷积设计中最基础、也最常用的关系式。无论是在软件框架中核对 layer shape，还是在 FPGA 上分配缓冲区、估算访存规模，本质上都离不开这个公式。

书里一个很重要的切入点是 `im2col`。它的思想并不复杂：把输入特征图上所有会被卷积核覆盖到的局部窗口，逐个拉平成列向量，再拼成一个大矩阵；同时把卷积核也拉平成向量。这样一来，原本的卷积计算就能重写为**矩阵乘法**。

从实现角度看，`im2col` 的优点和代价都很明显：

- 优点是计算结构规整，非常适合映射到 GEMM 风格的数据通路。
- 代价是中间展开矩阵会显著增大存储和搬运开销。

这也是很多 FPGA CNN 加速器都会面对的一个经典权衡：究竟是选择更规则、更容易并行的 `im2col + matrix multiply`，还是选择更节省片上存储的 direct convolution 数据流。

## 1.2 反卷积算子

反卷积也常被称为**转置卷积**或**上采样卷积**。书里强调了一点，这一点我觉得很重要：**反卷积不是正向卷积的严格逆运算**。它本质上仍然可以理解成一种特殊形式的正向卷积，只是输入和填充方式发生了变化。

直观上可以这样理解：

- 它服务的目标通常不是压缩空间尺寸，而是把低分辨率特征映射回高分辨率空间。
- 当步长大于 1 时，需要在输入像素之间插入额外的 0。
- 卷积核和边界填充方式也会随之变化。

书中给出的一个关键公式是反卷积输出尺寸：

$$
O' = S(I' - 1) + K - 2P
$$

这个公式在做检测、分割或生成类网络加速时非常常见。因为这类网络往往不仅关心“算得快”，还关心**输出 feature map 的空间尺寸是否精确对齐**。如果 shape 推导有误，后续层很容易直接对不上。

## 1.3 池化算子

池化本质上是一个降采样过程。书里主要整理了两种最常见的池化：

- 平均池化：输出 patch 内数值的平均值。
- 最大池化：输出 patch 内的最大值。

这类算子在数学上不复杂，但在硬件实现里很“干净”：

- 最大池化更像比较器树。
- 平均池化更像加法归约后再做缩放。

它们的共同特点是**参数量几乎没有，但会改变特征图尺寸与数据流节奏**。所以池化虽然不像卷积那样占据绝对主导的 MAC 数量，却会明显影响缓冲区划分、流水线边界以及跨层调度策略。

## 1.4 激活算子

激活函数的核心价值在于为网络引入非线性。如果没有激活函数，多个线性层叠加之后仍然等价于一个线性变换，模型表达能力会严重受限。

从硬件视角看，激活函数是一个非常典型的“**算法上很自然，电路上要看实现成本**”的模块：

- `ReLU` 非常适合硬件，因为本质上只需要比较和截断。
- `Sigmoid`、`Tanh`、`Softmax` 这类涉及指数、除法或归一化的函数，实现代价明显更高。

这也是为什么很多 FPGA 加速器更偏好 ReLU 或其变种，而更复杂的非线性函数则常常需要查表、分段逼近，或者干脆挪回软件侧处理。

## 1.5 全连接算子

全连接层可以看成一个标准的矩阵乘向量问题。当前一层卷积输出多个通道的特征图时，常见做法是先把它们全部拉平，变成一个一维向量，再与全连接层的权重矩阵相乘。

如果前一层输出 `100` 个 `4 x 4` 特征图，那么拉平之后向量长度就是：

$$
N = 100 \times 4 \times 4 = 1600
$$

之后全连接层的权重矩阵大小会由“输入长度 `N`”和“输出类别数 `T`”共同决定。

这一层在现代 CNN 中占比可能已经不像早期网络那么夸张，但它仍然是理解**“神经网络推理 = 大量线性代数运算”**的最好例子之一。对于 FPGA 来说，全连接层往往意味着：

- 数据复用模式相对简单；
- 访存模式较规则；
- 当参数量特别大时，带宽和片外存储压力会非常明显。

## 1.6 Softmax算子

Softmax 通常位于分类网络的最后一层，它的作用是把一组原始分数归一化为概率分布。书里给出的重点不是推导本身，而是它的功能定位：**让输出结果可以直接解释为“属于各个类别的概率”**。

从硬件实现角度看，Softmax 属于不太“友好”的算子，因为它涉及：

- 指数运算；
- 累加求和；
- 归一化除法。

这意味着它不像卷积、池化那样天然适合大规模流水并行。因此在很多实际系统里，Softmax 不一定是优先做硬件加速的对象，尤其是在只关心 top-1 / top-k 分类结果时，甚至可能直接比较 logits 而不显式做完整 Softmax。

## 1.7 批标准化算子

BN（Batch Normalization）是书里我觉得讲得比较值得保留的一节，因为它很清晰地点出了 BN 的本质用途：**稳定中间层输入分布，加快训练收敛，并缓解深层网络训练中的梯度问题**。

它通常放在激活函数之前，对线性变换 `y = Wx + b` 的输出做标准化，使结果满足特定均值和方差分布。书中把 BN 的过程拆成若干步骤来理解，这对初学者很友好。

但从推理部署角度，我更想补一句：在真正做 FPGA 推理实现时，BN 往往不会以“独立层”的形式完整保留，而是常常被**折叠进卷积或全连接层参数**中。这样既能减少运行时开销，也能简化数据路径。这一点虽然书里没有展开，但恰恰是从“理解算子”走向“理解部署”的关键一步。

## 1.8 Shortcut算子

Shortcut 的意义并不在于它本身计算多复杂，而在于它改变了深层网络的可训练性。书里给出的解释很直白：Shortcut 的存在可以让梯度有一条更稳定的传播路径，从而缓解深层网络中的退化问题。

如果把它放到残差网络（ResNet）里理解，就更容易明白：

- 主分支负责学习残差映射；
- 快捷分支负责提供恒等或近恒等的信息通路；
- 如果某一层没有必要，网络理论上可以把主分支学成接近 0。

从硬件视角看，Shortcut 最大的意义不是增加了多少计算，而是它引入了**跨分支的数据同步与逐元素相加**。这会影响：

- 缓冲区组织；
- 分支对齐；
- 中间结果保存策略；
- 多算子流水的时序边界。

也就是说，Shortcut 是一个非常好的例子，它提醒我们：**神经网络硬件实现的难点，不只在单个算子本身，还在算子之间的数据依赖与图结构。**

## 1.9 这一章值得带走的重点

如果把这一章压缩成几条真正有用的复习要点，我会留下下面这些：

- 卷积、池化、全连接本质上都可以归结为规则的数据搬运与线性计算，只是复用方式不同。
- `im2col` 让卷积计算更规整，但会带来额外的数据展开成本。
- 反卷积不是“逆卷积”，而是为了上采样服务的一类特殊卷积操作。
- 激活、Softmax、BN 这类算子虽然 MAC 数量不一定最大，但常常决定实现复杂度和部署方式。
- Shortcut 说明神经网络硬件实现不能只看单层，还必须看网络图结构和跨层数据流。

## 1.10 我的评价

如果只评价这本书的这一章，我认为它的价值主要在于：**帮初学者把常见神经网络算子和 FPGA 加速问题建立起第一层联系**。它适合作为“有一个总览”的材料，但不太适合作为真正落地实现的唯一参考。

我自己的感受是，这本书更适合在下面两种情况下阅读：

- 你刚开始接触“神经网络上板加速”这个方向，想先建立基本术语和算子框架。
- 你已经知道卷积、池化、BN 这些概念，但希望从 FPGA 实现的角度再看一遍它们的组织方式。

如果目标是进一步做完整工程、理解模块划分、读懂 RTL 或搭建可运行原型，那么还需要继续找更偏实现细节的资料。但作为一篇读书笔记，这一章确实值得被单独整理出来。
