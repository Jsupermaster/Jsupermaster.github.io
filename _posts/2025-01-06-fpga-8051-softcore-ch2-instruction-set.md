### 2.8051的指令集

8051指令集是非定长的，有一字节、二字节和三字节三种，其中第一个字节是指令标识，后面的字节是地址或立即数。8051指令集中共包含111条指令。

#### 2.1 指令的寻址方式

指令的寻址方式指的是指令如何提供地址，找到其要操作的对应数据的问题。

立即寻址：立即数不保存在数据池中，而是保存在指令的某个字节中。

直接寻址：直接寻址表示所想访问的数据池地址存在于指令的后一字节上，可以目的/源。

寄存器寻址：在RS段寻找数据，PSW中的RS0\RS1提供2位的地址，用于在DATA区确认使用32字节，4组中的哪一组。当PSW指示的某一组生效，则可以使用R0\~R7指示这一组中的8个寄存器。使用时需要提供R0~R7即3bit的地址，然后从对应的Rn中取出数据。

寄存器间接寻址：在寄存器中存放操作数的地址而非操作数，这时需要从寄存器中读出地址，再用这个地址从数据池读出数据。需要注意的是，寄存器间接寻址只允许使用最低2位寄存器，记作Ri，i=0或1.同样的，使用哪一组的Ri由PSW寄存器中的R1\R0确认。

变址寻址：访问CODE和XDATA需要16bit地址，使用变址寄存器和基址寄存器地址相加得出。ACC用作变址寄存器，PC或DPTR用于基址寄存器。

相对寻址：以PC为基准，只需提供偏移量即可获得CODE区地址

位寻址：20H\~2FH的区域、SFR区以0，8结尾的地址可进行位寻址，指令的[7]位用于确定区域，0代表20H\~2FH，1代表SFR区，[6:3]确定哪个字节，[2:0]确定哪一位。

#### 2.2 指令分类详解

![微信图片_20241228200745](/assets/images/fpga_8051_softcore/微信图片_20241228200745.jpg)

位置在0xA5上的是一条位置指令，可以用作拓展指令。位置0x00上的NOP指令是空指令，唯一的作用是让PC＋1。有一些格中的助记符一致，说明指令功能类似。

##### 2.2.1 算术操作指令

- ADDC ：带进位的加法指令，共4条
- SUBB ：减法指令，共4条
- INC ：加1指令，共5条

在这里我们以ADD为例，详细解析几种寻址方式的不同，后面的其他指令与此类似，在此不再赘述，书中包含更详细的解释。

**ADD A，Rn**

指令构成为{0010_1xxx}，单字节。

这条指令使用寄存器寻址，后面的“xxx”也即指令的[2:0]3位指示了Rn究竟是R0\~R7中的哪个，从这个寄存器中读出数据，将其与ACC中的数据相加，结果写回ACC中。

其verilog判别函数是：

```verilog
function add_a_rn(input[7:0] i);
    add_a_rn = (i[7:3] == 5'b00101);
endfunction
```

**ADD A，Direct**

指令构成为&#123;&#123;0010_0101},{direct address&#125;&#125;，双字节

这条指令使用直接寻址，第二字节的direct address是数据池中的地址，从这个地址找到数据，将这个数据与ACC中的数据相加，结果写回ACC中。

其verilog判别函数是：

```verilog
function add_a_di(input[7:0] i);
    add_a_rn = (i == 8'b00100101);
endfunction
```

**ADD A，@Ri**

指令构成为{0010_011i}

这条指令采用寄存器间接寻址，寄存器间接寻址只能利用R0和R1，所以指令中只有一位i，判断是R0还是R1。从指示的寄存器中获取地址，按照这个地址从数据池中寻找数据，将取出的数据与ACC中的数据相加，结果写入ACC中。

其verilog判别函数是：

```verilog
function add_a_ri(input[7:0] i);
    add_a_rn = (i[7:1] == 7'b0010011);
endfunction
```

**ADD A，#Data**

指令构成&#123;&#123;0010_0100},{Data&#125;&#125;

这条指令采用立即寻址，指令的第二个字节存放的就是操作数，将这个数与寄存器ACC中的数据相加，结果写入ACC中。

其verilog判别函数是：

```verilog
function add_a_da(input[7:0] i);
    add_a_da = (i == 8'b00100100);
endfunction
```

##### 2.2.3 数据转移类指令

即MOV类指令，主要控制数据在数据池中的移动。

- MOV:移动指令（16条）
- MOVC：从CODE区移动数据指令（2条）

##### 2.2.5 程序跳转指令

程序跳转指令的作用是改变PC，使程序跳转到新的地址上。这些指令使以字节为基础的指令，上一节中也有一些以位为基础的跳转指令。

- ACALL ：绝对调用指令（1条）
- LCALL ：长调用指令（1条）
- RET ：程序返回指令（1条）
- RETI ：中断返回指令（1条）
- AJMP ：绝对跳转指令（1条）
- LJMP ：长跳转指令（1条）
- SJMP ：短跳转指令（1条）
- JMP ：跳转指令（1条）
- JZ : ACC 的值等于0时跳转指令（1条） 
- JNZ : ACC 的值不等于0时跳转指令（1条）
- CJNE ：比较两字节不等跳转指令（4条）
- DJNZ ：字节减1并判断其值不等于0时跳转指令（2条）
- NOP ：空指令（1条）

其中，前4种属于CALL（调用）类指令，与JUMP（跳转）类指令不同的是CALL类需要保存下一条指令的地址（保存现场、恢复现场）

ACALL：绝对调用，2字节，识别序列10001

ACLL的组成是：{a10-18,{10001},{a7-a0&#125;&#125;，故第一字节的高3位和第二字节组成绝对地址a[10:0]。

以ACALL为例，执行该指令分为以下几步：

```
PC=PC+2	;PC现在指向当前指令，该指令是2字节，故＋2以指向下一条指令
SP=SP+1	;SP栈顶地址+1
(SP)=PC[7:0]
SP=SP+1
(SP)=PC[15:8]	;PC长度是16bit，分高低位存入栈中
PC[10:0]=a[10:0]	;将指令中包含的绝对地址送入PC中实现绝对调用
```

LCALL：长调用，3字节，识别序列00010010

LCALL的组成是&#123;&#123;0001_0010},{addr[15:8]},{addr[7:0]&#125;&#125;，第一字节是识别码，后面两个字节是长地址，其调用方法与上面的ACALL一致，也需要通过堆栈保存下一条指令地址，只不过其将a[15:0]赋值给PC。

RET：程序返回，1字节，识别序列00100010

RET是与LCALL,ACLLL相反的作用，从堆栈中取出一条指令地址，并返回这条指令执行程序。

RETI：功能与RET相同，但RETI用于中断服务程序之后使指令返回原地址，执行完这条指令后需要对中断的硬件寄存器管理。

跳转指令列举如下：

| 类型 | 说明                                                         | 字节 | 识别序列 |
| ---- | ------------------------------------------------------------ | ---- | -------- |
| AJMP | 绝对跳转，a[10:0]->PC[10:0]                                  | 2    | 00001    |
| LJMP | 长跳转，新地址送入PC                                         | 3    | 00000010 |
| SJMP | 短跳转（相对地址跳转）                                       | 2    | 10000000 |
| JMP  | 无条件跳转指令，PC=DPTR+ACC                                  | 1    | 01110011 |
| JZ   | ACC等于0跳转，PC＋第二字节                                   | 2    | 01100000 |
| JNZ  | ACC不等于0跳转，PC＋第二字节                                 | 2    | 01110000 |
| CJNE | 第二字节直接寻址，该地址的值与ACC不相等跳转，PC＋第三字节<br />ACC小于直接寻址数据值，PSW的CY置1，否则置0 | 3    | 10110101 |
| CJNE | 第二字节立即寻址，该地址的值与ACC不相等跳转，PC＋第三字节<br />ACC小于直接寻址数据值，PSW的CY置1，否则置0 | 3    | 10110100 |
| CJNE | 第二字节立即寻址，该地址的值与Rn不相等跳转，PC＋第三字节<br />ACC小于直接寻址数据值，PSW的CY置1，否则置0 | 3    | 10111xxx |
| CJNE | 第二字节立即寻址，该地址的值与Ri寄存器间接寻址不相等跳转，PC＋第三字节，ACC小于直接寻址数据值，PSW的CY置1，否则置0 | 3    | 1011011i |
| DJNZ | 寄存器寻址找到Rn，Rn减1，如果不等于0执行跳转，新地址PC＋第二字节 | 2    | 11011xxx |
| DJNZ | 第二字节直接寻址找到数据，数据减1，如果不等于0执行跳转，PC＋第三字节 | 3    | 11010101 |
| NOP  | 空指令，跳转到下一条指令执行                                 | 1    | 00000000 |

#### 2.3 指令执行对PSW的影响

PSW是状态寄存器，指令执行过程中不仅会改变数据池的内容，也会改变状态寄存器的值，便于后面的指令了解之前的执行状态。

![文档扫描_20241227221651214](/assets/images/fpga_8051_softcore/文档扫描_20241227221651214.jpg)

bit[1]和[5]都是用户可以自定义的状态位，它们和RS0\RS1都不会在指令执行时自动改变。RS1\RS0用于在寄存器组区中选择不同的分组，以便访问Rn和Ri。其他4位，CY，AC，OV，P都会在指令执行过程中自动改变。

P位指示的是ACC中1的个数，1的个数为偶数，P为0，否则P为1.

![文档扫描_20241229233709567](/assets/images/fpga_8051_softcore/文档扫描_20241229233709567.jpg)

ADD、ADDC、SUBB会同时影响CY、OV、AC；

CY是加减法的进位，两个8bit的数据相加可能溢出一位变成9bit，将第9位写入CY；

OV是相加减溢出标志位，对于有符号数来说，低7位是数据，第8位是符号，7bit的进位可能导致符号位发生错误。OV定义如下：如果8bit加减的结果没有产生进位，那么若低7bit加减产生进位，则对OV置1，否则写入0；如果8bit加减结果产生进位，那么若低7bit加减产生进位，则对OV置0，否则置1。

AC标志是低4bit加减时的进位。

以上所述内容的verilog实现：

```verilog
wire 	[7:0]	a,b								;//a,b是参与加减法运算的字节
wire			c								;//c是参与加减法运算的进位bit
wire			sub_flag						;//减法运算标志
wire			bit_ac,bit_c,bit_ov,bit_high	;//AC标志、CY标志、OV标志、结果运算最高位
wire	[3:0]	low,high						;//中间运算
wire	[7:0]	byte							;//最终结果

//低4bit加减法，产生bit_ac
assign {bit_ac,low} = sub_flag ? {a[3:0]-b[3:0]-c} : {a[3:0]+b[3:0]+c};

//中间3bit加减法，得到低7bit进位
assign high = subflag ? {a[6:4]-b[6:4]-bit_ac} : {a[6:4]+b[6:4]+bit_ac};

//最高1bit加减法，得到bit_c
assign {bit_c,bit_high} = sub_flag ? {a[7]-b[7]-high[3]} : {a[7]+b[7]+high[3]};

//得到bit_ov
assign bit_ov = bit_c ^ high[3];

//得到最终加减法结果
assign byte = {bit_high,high[2:0],low};
```

除了加减法以外，同时影响CY和OV的是MUL和DIV：

MUL指令对CY写入0，如果16bit乘法结果的高8位不等于0，对OV写1，否则写0；

DIV指令对CY写入0，如果除数B寄存器是0，OV标志写1，否则写0.
