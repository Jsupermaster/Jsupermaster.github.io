## 2.1 硬件架构修改

在第一章中我们搭建了支持 RV32GC 的裸机运行 CPU，这是为了初学时便于理解。进一步为了启动 Linux 系统，我们需要对这个硬件架构进行升级。核心的升级有以下几点：

1. 生成设备树，修改总线和外设以适配系统加载。
2. 修改指令集架构为 RV64，更易于 Linux 的移植。
3. 修改内存、存储大小，确保能够满足系统运行。
4. 修改任务负载为加载系统。

为了便于日后代码复用和修改，以后的代码使用函数形式。

### 2.1.1 设备树生成

设备树（DTB/DTS）是 Linux 内核在启动时用来识别硬件拓扑结构的规范。由于 Gem5 中的硬件配置是动态组合的，比如内存大小、CPU 核心数可变，因此必须在模拟启动前动态生成对应的设备树文件。

首先定义内存设备树节点生成函数：

```python
def generate_mem_node(state, mem_range):
    node = FdtNode(f"memory@{int(mem_range.start):x}")
    node.append(FdtPropertyStrings("device_type", ["memory"]))
    node.append(
        FdtPropertyWords(
            "reg",
            state.addrCells(mem_range.start) + state.sizeCells(mem_range.size()),
        )
    )
    return node
```

**功能定位：** 为指定的物理内存范围生成一个设备树的 `memory` 节点。

**详细解析：**

- **`node = FdtNode(f"memory@{int(mem_range.start):x}")`**
  - 创建设备树节点，命名格式符合设备树规范：`节点名@起始地址（十六进制）`。例如，如果内存从 `0x80000000` 开始，节点名就是 `memory@80000000`。
- **`node.append(FdtPropertyStrings("device_type", ["memory"]))`**
  - 为该节点添加属性 `device_type = "memory"`。这是 Linux 内核识别内存节点的标准标志。
- **`node.append(FdtPropertyWords("reg", state.addrCells(mem_range.start) + state.sizeCells(mem_range.size())))`**
  - 配置关键的 `reg` 属性。`reg` 描述了该设备的地址和大小。
  - `state.addrCells()` 和 `state.sizeCells()` 根据全局状态 `state` 把起始地址和大小转换成设备树规定的字长格式。这两部分被拼接在一起填入 `reg` 属性中。
- **`return node`**
  - 返回组装好的内存节点对象，供后续挂载到设备树根节点。

之后定义完整的设备树生成函数：

```python
def generate_dtb(system):
    state = FdtState(addr_cells=2, size_cells=2, cpu_cells=1)
    root = FdtNode("/")
    root.append(state.addrCellsProperty())
    root.append(state.sizeCellsProperty())
    root.appendCompatible(["riscv-virtio"])

    for mem_range in system.mem_ranges:
        root.append(generate_mem_node(state, mem_range))

    for section in [*system.cpu, system.platform]:
        for node in section.generateDeviceTree(state):
            if node.get_name() == root.get_name():
                root.merge(node)
            else:
                root.append(node)

    fdt = Fdt()
    fdt.add_rootnode(root)
    fdt.writeDtsFile(path.join(m5.options.outdir, "device.dts"))
    fdt.writeDtbFile(path.join(m5.options.outdir, "device.dtb"))
```

**功能定位：** 核心编排函数，负责收集系统中所有的硬件信息，生成完整的设备树，并将其写入文件。

**详细解析：**

- **`state = FdtState(addr_cells=2, size_cells=2, cpu_cells=1)`**
  - 初始化设备树的全局状态。
  - `addr_cells=2` 和 `size_cells=2` 表示物理地址和大小都占用 2 个 32-bit 单元，即 64 位。这表明这是一个 64 位的 RISC-V 系统。
  - `cpu_cells=1` 表示 CPU 的硬件 ID 用 32 位表示即可。
- **`root = FdtNode("/")`** 及属性追加：
  - 创建设备树的根节点 `/`。
  - 将全局的地址和大小单元格属性挂载到根节点。
  - `root.appendCompatible(["riscv-virtio"])` 声明该系统的兼容性标志为 `riscv-virtio`，内核会根据这个标志加载相应的平台驱动。
- **内存节点挂载**：
  - 遍历 `system.mem_ranges`，调用 `generate_mem_node` 生成所有内存范围的节点，并添加到根节点中。
- **CPU 及外设节点挂载**：
  - `for section in [*system.cpu, system.platform]` 遍历系统中的所有 CPU 核心和平台外设，如中断控制器、串口、VirtIO 设备等。
  - 调用它们自带的 `generateDeviceTree(state)` 方法生成对应的子设备树。
  - 如果某个组件生成的节点名字也是 `/`，则使用 `root.merge(node)` 进行合并；否则，将其作为子节点追加到根节点下。
- **生成并保存文件**：
  - 创建 `Fdt` 对象，绑定根节点。
  - `fdt.writeDtsFile(...)` 会在 gem5 的输出目录中生成 `.dts` 文件，便于开发者调试检查硬件配置。
  - `fdt.writeDtbFile(...)` 会生成 `.dtb` 文件，这是真正传给 bootloader 和 Linux 内核读取的文件。

### 2.1.2 修改硬件架构代码

**内存容量与地址修改**

```python
# Origin
system.mem_ranges = [AddrRange("512MB")]
# Now
system.mem_ranges = [AddrRange(start=0x80000000, size="2GB")]
```

**指令集架构修改**

```python
# Origin
system.cpu = RiscvTimingSimpleCPU()
system.cpu.isa = [
    RiscvISA(
        riscv_type="RV32",
    )
]
# Now
system.cpu = [RiscvTimingSimpleCPU(cpu_id=0)]
system.cpu[0].isa = [RiscvISA(riscv_type="RV64")]
```

**总线修改**

```python
system.membus = SystemXBar()
system.iobus = IOXBar()

system.cpu[0].icache_port = system.membus.cpu_side_ports
system.cpu[0].dcache_port = system.membus.cpu_side_ports
system.cpu[0].mmu.connectWalkerPorts(
    system.membus.cpu_side_ports, system.membus.cpu_side_ports
)
system.cpu[0].createInterruptController()
system.system_port = system.membus.cpu_side_ports
```

相较于之前，主要是新增了 `iobus`，以及新增了 MMU 的连接。

**硬件平台修改**

```python
system.platform = HiFive()
system.platform.uart.device = system.platform.terminal
system.platform.rtc = RiscvRTC(frequency=Frequency("100MHz"))
system.platform.clint.int_pin = system.platform.rtc.int_pin
system.platform.setNumCores(1)
system.platform.attachPlic()
system.platform.pci_host.pio = system.iobus.mem_side_ports
```

将串口连接到终端，不再输出到文件，确保可以进行交互；新增 RTC（实时时钟），并连接到中断器端口，处理定时器中断。

挂载 PLIC 来处理全局外部中断；将 PCI 宿主控制器的可编程 I/O 接入 `iobus`。

**新增磁盘挂载**

```python
image = CowDiskImage(child=RawDiskImage(read_only=True), read_only=False)
image.child.image_file = ROOTFS_IMAGE
system.platform.disk = RiscvMmioVirtIO(
    vio=VirtIOBlock(image=image),
    interrupt_id=0x8,
    pio_size=4096,
    pio_addr=0x10008000,
)
```

磁盘镜像保护：底层的 `ROOTFS_IMAGE` 被设置为只读。在它之上套了一层 `CowDiskImage`，这样 Linux 启动后所有的写入操作都会保存在临时层中，不会破坏原始镜像文件。

VirtIO 挂载：创建一个 VirtIO 块设备作为系统的 `/dev/vda` 磁盘，将其映射到 MMIO 地址 `0x10008000`，并分配中断号 `0x8`。

**新增双向桥连接**

```python
# CPU -> MMIO bridge
system.bridge = Bridge(delay="50ns")
system.bridge.mem_side_port = system.iobus.cpu_side_ports
system.bridge.cpu_side_port = system.membus.mem_side_ports
system.bridge.ranges = system.platform._off_chip_ranges()

# IO DMA -> DRAM bridge (needed by VirtIO data path)
system.iobridge = Bridge(delay="50ns", ranges=system.mem_ranges)
system.iobridge.cpu_side_port = system.iobus.mem_side_ports
system.iobridge.mem_side_port = system.membus.cpu_side_ports
```

系统在这两条总线之间建立“双向车道”，并设置 50ns 的延迟来模拟真实的物理损耗：

- `bridge`：CPU 访问外设时使用。当 CPU 想读写寄存器时，指令从 CPU 发到 `membus`，通过这个桥转发到 `iobus`。
- `iobridge`：外设访问主存时使用，也就是 DMA 操作。例如，VirtIO 磁盘把数据读出来后，直接通过 `iobus` 穿过这个桥写入主存，不需要 CPU 一点点搬运。

**修改负载并启动设备树**

```python
system.workload = RiscvLinux()
system.workload.object_file = BBL_PAYLOAD
system.workload.command_line = "console=ttyS0 root=/dev/vda rw rootwait"

generate_dtb(system)
system.workload.dtb_filename = path.join(m5.options.outdir, "device.dtb")
system.workload.dtb_addr = 0x87E00000
```

Linux 负载：声明这是一个 Linux 系统。将前面的 `BBL_PAYLOAD` 设为启动对象。

内核参数：传递启动参数 `console=ttyS0` 和 `root=/dev/vda rw rootwait`，告诉 Linux 根文件系统位于刚才配置的 VirtIO 磁盘上，且支持读写。

设备树加载：调用前面提供的 `generate_dtb` 函数，根据当前配置动态生成设备树文件，并将其加载到内存地址 `0x87E00000` 供 Linux 内核在启动时读取。

## 2.2 获取Bootloader和Kernel

我们直接使用 Gem5 仓库准备好的 RISC-V Linux 启动镜像：

```bash
# 1) bootloader+kernel 打包文件（给 RiscvLinux().object_file）
curl -L "https://dist.gem5.org/dist/v22-1/kernels/riscv/static/bootloader-vmlinux-5.10" \
  -o myriscv/images/bbl_payload
# 2) rootfs 镜像
curl -L "https://dist.gem5.org/dist/v22-1/images/riscv/busybox/riscv-disk.img.gz" \
  -o myriscv/images/rootfs.img.gz
gunzip -f myriscv/images/rootfs.img.gz
```
